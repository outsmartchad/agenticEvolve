# Agent Publish Gate

A pre-publish review layer for agent-generated content. Prevents autonomous agents from publishing unreviewed content to external channels — the pattern failure behind the HN-viral "AI agent published a hit piece on me" incident (2346 upvotes, 2026-03).

## When to Use

- Before any agent action that sends content outside the local environment
- Reviewing agenticEvolve Telegram/WhatsApp messages generated by the evolve cycle
- Before a GitHub issue, PR comment, or blog post is created by an autonomous agent
- Adding a human-in-the-loop gate for content with reputational risk

## The Incident Pattern

An autonomous agent researched a public figure, formed an opinion, and published a blog post without human review. The post was factually wrong and reputationally damaging. The root cause: no publish gate between agent reasoning and external action.

## Implementation: Pre-Publish Hook

Add to `~/.claude/settings.json`:

```json
{
  "hooks": {
    "PreToolCall": [
      {
        "matcher": "mcp__.*__.*send.*|Bash",
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/hooks/publish-gate.sh"
          }
        ]
      }
    ]
  }
}
```

`~/.claude/hooks/publish-gate.sh`:
```bash
#!/bin/bash
# Intercept publish-capable tool calls and require confirmation
INPUT=$(cat)
TOOL=$(echo "$INPUT" | jq -r '.tool_name // ""')
PUBLISH_TOOLS=("sendMessage" "createIssue" "createPR" "postTweet" "publishPost")

for t in "${PUBLISH_TOOLS[@]}"; do
  if echo "$TOOL" | grep -qi "$t"; then
    echo "PUBLISH GATE: Tool '$TOOL' requires human review before proceeding." >&2
    echo "Content preview:" >&2
    echo "$INPUT" | jq -r '.tool_input | to_entries[] | "\(.key): \(.value)"' | head -10 >&2
    exit 1  # Block — human must re-run with explicit approval
  fi
done
exit 0
```

## agenticEvolve Integration

For the Telegram notification at end of evolve cycle, add a review step:

```python
# gateway/evolve.py — REPORT stage
REPORT_PROMPT = """
Review the following evolve cycle summary before sending to Telegram.
Check for:
- Any claims about real people or organizations that could be defamatory
- Factual assertions that are not backed by the signal sources
- Tone that is aggressive or could embarrass the sender

If any red flags: return JSON {"approved": false, "reason": "..."}
Otherwise: return JSON {"approved": true, "content": "<final message>"}
"""
```

## Checklist

- [ ] Hook installed and executable: `chmod +x ~/.claude/hooks/publish-gate.sh`
- [ ] Tool matcher covers all external-send MCP tools in use
- [ ] agenticEvolve REPORT stage includes a content review step
- [ ] Test: verify hook blocks a test sendMessage call before deploying
- [ ] Document which publish actions are exempt (e.g. git commits are local-only)

## Rules

- Never bypass this gate with `--no-verify` or by removing the hook temporarily
- Exempted tools (local file writes, git commits) must be explicitly listed — default is block
- Gate applies to all agents including automated cron jobs
